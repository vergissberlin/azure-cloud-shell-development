# Contributing

## Development Workflow

1. Create a feature branch.
2. Implement changes with minimal, focused scope.
3. Validate scripts and docs locally.
4. Open a pull request with clear rationale and test notes.

## Project Conventions

- Keep shell code portable and readable.
- Prefer user-space install behavior for Cloud Shell compatibility.
- Avoid assumptions about root access.
- Keep docs in English and update relevant pages with behavior changes.

## Versioning Rules

- Keep `SCRIPT_VERSION` in sync with `.release-please-manifest.json`.
- Follow existing release automation conventions.
